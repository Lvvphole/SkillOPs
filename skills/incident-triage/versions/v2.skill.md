# Incident Triage Skill — v2 (candidate template)

Same urgent/standard policy as v1, with a documented slot for evidence-backed
refinements produced by the corrective or optimization branch.

## Policy
- urgent  := severity == "high" OR customer_impact == true
- standard := otherwise
- action  := taken when label == urgent

## Change rationale (must be evidence-backed)
A v2 is only adopted when:
1. every required threshold from `thresholds.yaml` is preserved, and
2. at least one approved optimization metric (`false_positive_rate`,
   `avg_duration_ms`) improves versus v1 on the same evidence sample.

Until that evidence exists, v2 remains a template and the loop records a
no-change decision. This file documents the candidate shape; it does not change
the active version on its own (see `ACTIVE_VERSION` / `rollback_skill`).
